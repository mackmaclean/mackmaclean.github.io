Feedback - ASK: Actionable, Specific, Kind

Practice writing kind, actionable, specific feedback to other students.

Here are some sample questions to use:

  • If you were going to implement the pseudocode, would it be easy to follow?
  • Proper Variable Naming - make the variable name descriptive and concise - Avoid naming variables after the data structure (array, string, arr, hash, hsh, etc) - Avoid numbered variables (var1, var2, var3) - Avoid single character names (i, x, n)
  • Proper Method Naming *The name should reflect the method's single responsibility
  • Keep it DRY - Condense repetitive code - Create new methods to encapsulate common functions
  • Did you like the logic?
  • Was every variable necessary)
  • What did the student do well?
  • How was their reflection?
  • Did you learn anything from this solution?
