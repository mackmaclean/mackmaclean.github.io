GPS 1.2 - Pair Programming

What is pair programming?

Pair programming is a software development technique in which two programmers work together at one work station. One (driver) writes code while the other (navigator) reviews each line of code as it is typed. The two programmers should switch roles frequently.

Why is pair programming important to DBC?

Pair programming in DBC is very beneficial as it helps with learning new material.

Compare driver and navigator

Driver types code while the navigator observes, makes suggestions, find errors, and corrects syntax, while taking an overview perspective.

Reflection

Peer Paring is a good opportunity to meet new people, learn from each other, build relationships, and to stay on-track :).
